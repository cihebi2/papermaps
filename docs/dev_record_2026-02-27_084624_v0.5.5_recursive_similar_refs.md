## 7. 接口契约（如有）
- 接口名称：`POST /api/works/recursive-similar-references`
- 用途：按层递进扩展参考文献，并用 seed 摘要相似度筛选每层候选。
- 输入参数：
  - `doi` / string / 必填 / seed DOI（支持 `https://doi.org/...` 或裸 DOI）
  - `depth` / int / 选填 / 默认 3 / 递进层数（1-5）
  - `max_references` / int / 选填 / 默认 10 / 每个父节点最多考虑多少条参考文献（1-200）
  - `top_k` / int / 选填 / 默认 2 / 每个父节点选取相似度最高的前 K 个（1-50）
  - `min_score` / number / 选填 / 默认 0.08 / Jaccard 阈值（0-1）
  - `save` / bool / 选填 / 默认 true / 是否写入本地 DB（papers/edges/watch_targets）
- 返回值：
  - `ok: true`
  - `seed: {paper_id,title,doi,...}`
  - `layers: [{level:int, nodes:[{level,parent_paper_id,score,shared_terms,work:{...}}]}]`
  - `fetch_count, elapsed_ms` 等统计字段
- 错误处理：
  - 400：DOI/参数非法，或 seed 无可用摘要/标题。
  - 404：seed DOI 未找到。
  - 500：内部错误。
- 边界条件：
  - 某层选不出结果会提前停止（layers 长度可能 < depth）。
  - 单个引用 work 拉取失败会跳过并记录 warning。
- 幂等性：
  - `save=false` 时幂等。
  - `save=true` 时对同一边写入使用 ON CONFLICT DO NOTHING（不会重复插边）。
