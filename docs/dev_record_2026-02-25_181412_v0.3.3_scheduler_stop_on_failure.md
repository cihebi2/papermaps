7. 接口契约（如有）
- 接口名称：`run-scheduler` CLI 参数 `--stop-on-failure`。
- 用途：控制 scheduler 在单轮失败后是否立即停止。
- 输入参数：
  - `--stop-on-failure` / flag / 非必填 / 默认 false / 开启后遇到失败立即停止。
- 返回值：CLI 退出码；有失败时返回非 0。
- 错误处理：沿用现有 `track-citations` 失败语义。
- 边界条件：`iterations<=0`、`interval<0` 仍按原规则失败。
- 幂等性：同参数重复运行为可重复执行流程。
- 使用示例：`python -m src.cli run-scheduler --config config.yaml --iterations 3 --stop-on-failure`。
