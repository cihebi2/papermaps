# v0.5.7 开发记录（Web Recursive Loom UI Visualization）

时间戳：2026-02-27 17:20:11
版本：v0.5.7

## 1. 本步 DoD
- 功能完成条件：网页端 “Recursive Similar References” 的结果不再只输出文本，而是以可交互的“图谱/层级织网（Loom）”方式呈现：Seed/L1/L2/L3 分列、连线可视化、hover 高亮、click 详情、Raw JSON 展示、可清空重置。
- 验收方式：
  - 启动：`python -m src.cli serve-web --db-path data/papermap.db --host 127.0.0.1 --port 8765`
  - 打开：`http://127.0.0.1:8765/`
  - 在 “Recursive Similar References” 输入 DOI，点击 `Expand (depth)` 后能出现 Loom 视图（列 + 卡片 + 连线）。
  - hover 任一卡片：相邻节点与连线高亮；click 卡片：左侧详情面板展示 why（parent、score、shared_terms）。
  - `Raw JSON` 可展开/收起；`Redraw edges` 可重绘连线；`Clear` 恢复空态。
- 测试通过条件：既有 Python 单测全绿（不引入/不破坏后端接口）。
- 产物清单：`src/web_server.py`、本记录。
- 明确不做范围：不新增第三方前端依赖（无 D3/cytoscape 等），不改后端 API/DB schema，不实现新的打分/检索逻辑。

## 2. 假设与风险
- 假设：
  - 浏览器支持现代基础语法（ES6 级别：模板字符串、`const/let`、`for..of` 等）。项目此前已采用该级别语法，本步保持一致。
  - 递归 API 返回字段保持：`seed/layers[*].nodes[*].{parent_paper_id,score,shared_terms,work}`。
- 风险：
  - 视图绘制依赖 DOM 布局完成后读取 `getBoundingClientRect()`，在字体加载/窗口缩放时可能需要重绘。
- 最小化影响策略：
  - 增加 `Redraw edges` 手动重绘按钮，并在 `resize` 时自动重绘一次。
  - 不修改后端输出结构，确保回滚仅涉及前端 HTML/CSS/JS 片段。
